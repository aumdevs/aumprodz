# API Reference

## GET `/api/cta/whatsapp`

Registers CTA intent and redirects to WhatsApp.

Query params:

- `service`: service slug, for example `youtube-adsense`, `paginas-web`, `imagen-video`, `artista`.
- `button`: button identifier.
- `source`: page or campaign source.
- `utm_source`: optional campaign source.

If Supabase service credentials are missing, the route still redirects safely.

## POST `/api/webhooks/stripe`

Uses Stripe raw body and `stripe-signature` verification. Requires:

- `STRIPE_SECRET_KEY`
- `STRIPE_WEBHOOK_SECRET`

Stores events in `webhook_logs`. For ARTISTA annual subscriptions, it also
updates `artist_subscriptions` and moves `artist_profiles.status` from
`pending_payment` to `active_pending_verification` when Stripe reports an
active/trialing subscription.

## GET `/api/checkout/artist-annual`

Requires an authenticated artist account. If the artist already has an active
annual subscription, redirects to the dashboard instead of creating another
Checkout Session. Otherwise creates a Stripe Checkout Session in subscription
mode using:

- `STRIPE_SECRET_KEY`
- `STRIPE_PRICE_ARTIST_ANNUAL`

The session stores `user_id` metadata so the signed Stripe webhook can reconcile
the payment back into Supabase. If Stripe env vars are missing, the route returns
`503` without writing payment state.

## GET `/api/checkout/ep-release`

Requires an authenticated artist account. If the artist already has an unused
paid EP credit, redirects to billing instead of creating another Checkout
Session. Otherwise creates a Stripe Checkout Session in payment mode using
`STRIPE_PRICE_EP_RELEASE`. The webhook stores the result in `artist_payments`
with `product_key = ep_release`.

## GET `/api/checkout/album-release`

Requires an authenticated artist account. If the artist already has an unused
paid album credit, redirects to billing instead of creating another Checkout
Session. Otherwise creates a Stripe Checkout Session in payment mode using
`STRIPE_PRICE_ALBUM_RELEASE`. The webhook stores the result in `artist_payments`
with `product_key = album_release`.

## POST `/api/events/public`

Receives public browser events such as `page_view`, `service_detail_view`,
`artist_pricing_view` and `artist_checkout_click`. If Supabase service
credentials are missing in local development, it returns `204` and skips the
write; when Supabase is configured, insert failures return `503`.

## POST `/api/webhooks/sendpulse`

Requires `SENDPULSE_WEBHOOK_SECRET` and header `x-aum-webhook-secret`.

## POST `/api/webhooks/identity`

Didit destination URL for KYC status updates. Use:

- `DIDIT_WEBHOOK_SECRET`: the `secret_shared_key` generated by the Didit
  webhook destination.
- Didit signature headers: `X-Timestamp` plus `X-Signature-V2` preferred,
  with `X-Signature` and `X-Signature-Simple` accepted as fallbacks.

For backwards-compatible manual local tests only, the route also accepts
`x-aum-webhook-secret` or `?secret=` when Didit signature headers are absent.

## POST `/api/webhooks/sign`

signNow destination URL for contract signature status updates. Create a
signNow event subscription with this URL as `attributes.callback` and set
`attributes.secret_key` to the same value as `SIGNNOW_WEBHOOK_SECRET`.

The route verifies `X-SignNow-Signature` using HMAC-SHA256 over the raw request
body. For backwards-compatible manual local tests only, it also accepts
`x-aum-webhook-secret` or `?secret=` when `X-SignNow-Signature` is absent.

## POST `/api/legal/identity/session`

Authenticated artist endpoint that creates a Didit verification session and
returns the provider verification URL.

## POST `/api/legal/contracts/[id]/send`

Admin-only endpoint that sends an uploaded PDF contract to signNow and stores
the signing link on `artist_contracts`.

## POST `/api/uploads/create-presigned-url`

Requires an authenticated Supabase session. The release must belong to the
artist and be editable (`draft` or `needs_changes`). Audio/video use R2; artwork,
lyrics and documents use the private Supabase Storage bucket
`artist-small-files`.

Body:

```json
{
  "releaseId": "uuid",
  "fileType": "audio",
  "fileName": "song.wav",
  "contentType": "audio/wav",
  "sizeBytes": 123456
}
```

Returns `fileId`, `key`, `provider`, `signedUrl`, `uploadMethod` and
`expiresIn`.

## POST `/api/uploads/complete`

Marks a pending release file as uploaded after the client finishes the signed
upload.

Body:

```json
{
  "fileId": "uuid"
}
```

Writes `artist_files.upload` or `artist_files.replace` to `audit_logs`.

## GET `/api/uploads/files/[fileId]`

Creates a short-lived signed URL and redirects to it.

Query params:

- `action=download`: requires owner access or `artist_files.download`.
- `action=listen`: requires owner access or `artist_files.listen`, and only works
  for previewable file types.

Every successful access writes an audit log.
